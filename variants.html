Updated Wed 22 Apr 2015 • tags mongolian, scriptnotes

There is some confusion about which shapes should be produced by fonts for Mongolian characters. Most letters have at least one isolated, initial, medial and final shape, but other shapes are produced by contextual factors, such as vowel harmony.

Unicode has a list of standardised variant shapes, dating from 27 November 2013, but that list is not complete and contains what are currently viewed by some as errors.

The original list of standardised variants was based on 蒙古文编码 by Professor Quejingzhabu in 2000.

A new proposal was published on 20 January 2014, which attempts to resolve the current issues.

The other factor in this is what the actual fonts do. Sometimes they follow the Unicode standardised variants list, other times they diverge from it. Occasionally a majority of implementations appear to diverge in the same way, suggesting that the standardised list should be adapted to reality.

In this document I map the changes between the various proposals, and compare to various font implementations.

How to use the document

For each Mongolian script letter, this page shows graphics to indicate the shape of a character in a given context (described in the left column). The graphics are standardised, and do not represent the actual font glyph shape.

The columns to the left show the shapes expected in each of the three documents mentioned above. The columns to the right indicate the shapes produced by various fonts. See below for a key to the abbreviations used.

The background of the left-most column is red if there is a conflict between the shapes used for the various specifications and orange if only one or more font implementations disagree with the specifications.

The right column of the table contains a sequence of characters to allow you to check what glyphs are produced by fonts to which you have access. The vertical blue line at the bottom right of the window opens, when you click on it, to show a font control panel. When it is open, you can apply a specific font to the text in the right-hand column by selecting it from the top pull-down menu in this control panel.

If the font you want to test is not available in the list selector, click on 'Add a font to these lists' (found below the two list selectors in the control panel just mentioned) and enter the name of the font you want to add in the dialog box that appears. The font name will be added to the selection controls. Open the top list selector again, and click on the name of the font to apply it to the text in the right column. (Fonts you add to a list selector only stay on the list for the duration of the session.)

In the text column I use U+180A MONGOLIAN NIRUGU by default to establish the context for connecting characters. By clicking on the radio button in the font dialog box, you can change the connector to U+200D ZERO WIDTH JOINER. I experienced significant issues with ZWJ and the Mongolian Baiti font on Internet Explorer, so that is why the nirugu is used as the default. The results reported here for Mongolian Baiti are for display on IE, but with the nirugu to create the context.

Abbreviations and fonts tested

Q: Prof Quejingzhabu's document

USVL: Unicode Standardised Variants list, v7

NP: New proposal, Jan 2014

MB: Mongolian Baiti font. Currently version 5.4.1, in Internet Explorer 11 on Windows 8. (Note that a fair number of results will be different for Baiti 5.0.1 on Windows 7, and that the use of nirugu vs. zwj can also affect the outcome for version 5.0.1.)

NSM: Noto Sans Mongolian font

MWc: a collection of fonts from the same source that all behave in the same way, including the following: Mongolian White, Mongolian Writing, Mongolian Title, Mongolian Art. If any of these fonts behaves differently from the others, I will add a column for it, or describe the difference in notes.

fvs: Free variant selector.

is, in, m, f: isolate, initial, medial, final, respectively.

All fonts tested produce the 1st medial form described by Professor Quejingzhabu. The 2014 proposal moves that glyph to the 2nd medial form due to research by the Professor that indicates, that according to syllable-forming rules, the medial loop so long considered to be a vowel, UE (U+1826), was determined to be the semi-vowel WA (U+1838) instead.

The same goes for the 1st and 2nd final glyphs.

This means that the fonts tested will need to change their behaviour.
