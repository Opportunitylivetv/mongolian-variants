This document lists shapes used to represent characters in the Mongolian script and how they are mapped to the characters and contexts in which they are used. The document doesn't show you where a glyph shape would be produced by OpenType rules in a font. Just the standard positional shapes and those produced by variant selectors.

The data is based on DS01, an update to the Jan 2014 L2/14-031 proposal, which is not currently publicly available, and is still a moving target. The information was supplied by Greg Eck, and developed by him, Aaron Bell, and Andrew Glass.

how to use the table

Each possible shape is listed in the left-most column. An attempt is made to arrange similar shapes together.

In the middle column you find Unicode code point numbers for characters that use that shape in ordinary positional contexts. The code point is immediately followed by s, i, m or f, indicating isolated, initial, medial or final, respectively.

Code points are colour-coded as follows: Mongolian, black; Todo, brown; Sibe, green; Manchu, violet; Ali Gali, blue.

If you mouse over the code point number, the tooltip shows the name of the character. If you click on the code point number, it takes you to another document containing a list of the various shapes for that character, and how the character in that context is displayed in major fonts.

The third column works in the same way as the second, except that code points are also followed by a number, which indicates that the character has to be used with FVS1, FVS2 or FVS3 to produce that shape.
